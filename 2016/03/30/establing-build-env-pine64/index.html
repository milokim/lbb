Pine64: Establing a Build Environment

2016-03-30

Pine64 is a low-cost ARM64 SBC (Single Board Computer).
This page desribes how to set up a build environment for Pine64 kernel development by using Docker.
Many thanks to Andre Przywara who has been working Pine64 mainline. Please refer to his page.

Focus on Linux kernel development

This page could be helpful if you are someone who

  • Prefer mainline kernel to vendor BSP kernel
  • Want to build and test your own kernel quickly
  • Don’t care the rootfs, simple initrd gonna be OK

Flash prebuilt image onto microSD card

Prior to updating your own kernel image, please make the default microSD card image.
Please refer to ‘Quick start’ section in this page.

Kernel version

Prebuilt microSD image is v4.4. And you will build v4.5-rc1.
Please note that Pine64 kernel mainline work is an initial step and on going.

Docker for kernel development

To compile Pine64 kernel, it requires cumbersome procedures like downloading cross-compiler, getting mailine kernel, ramdisk and so on.
Docker is a good choice to isolate your environment and accelerate building job.
Don’t spend your time to set up environment. :)

Install a Docker

Please refer to the installation page if not installed.

Create a container for building Pine64 kernel

Please use this Dockerfile.
To create a container, type commands below.

$ wget https://raw.githubusercontent.com/milokim/build-pine64/master/Dockerfile
$ docker build -t pine64-kernel .

It will take some time about 10 ~ 15 min. Enjoy a cup of coffee.
Everything is done, you can get the container ID like below.

Successfully built d9f75cbe1006

Run the container

$ docker run -i -t d9f75cbe1006 /bin/bash

Please replace ‘d9f75cbe1006’ with your container ID.

I strongly recommend using your local directory inside the container. You can share files between host and the container.
Please run a Docker with the volume option, ‘-v’.
For example, 

$ docker run -v /home/yourname/devel:/home/devel/ -i -t d9f75cbe1006 /bin/bash

In this case, /home/devel/ is accessible only inside the container. Actual files exist in your local directory, /home/yourname/devel/.

Welcome! You are in the container. 

root@cac34d6b1857:/home/pine64/linux#

Build the kernel

Just run the build script. This script was created while building a container by using the Dockerfile. 

root@cac34d6b1857:/home/pine64/linux# ./build.sh

If kernel is built successfully, ‘kernel.img’ is generated. It contains the kernel with initrd. This file will be updated into your microSD card.

Update new kernel and DTB onto microSD card

Copy kernel.img and dtb files from Docker container to your local directory. And exit from the container. 

root@cac34d6b1857:/home/pine64/linux# cp kernel.img /home/devel/
root@cac34d6b1857:/home/pine64/linux# cp arch/arm64/boot/dts/allwinner/*.dtb /home/devel/
root@cac34d6b1857:/home/pine64/linux# exit

Now, you are back to real machine. To update a container’s changes, use docker commit command below. Please use your ID instead of ‘cac34d6b1857’. 

$ docker commit cac34d6b1857 pine64-kernel

Insert SD card and use ‘dd’ command. Please replace ‘sdx’ with your environment by checking fdisk or dmesg. 

$ cd /home/yourname/devel/
$ sudo dd if=kernel.img bs=1M of=/dev/sdx9
$ sudo dd if=sun50i-a64-pine64.dtb bs=1M of=/dev/sdx seek=21

Pine64 has separate disk partions in microSD, so kernel and DTB should be loaded in each location.
For Pine64+ board, please use sun50i-a64-pine64-plus.dtb instead of this dtb.

Boot your own kernel

To load new built kernel, please follow steps below in u-boot prompt. 

(U-boot prompt)
sunxi# run load_env
sunxi# run load_dtb
sunxi# run set_cmdline
sunxi# setenv kernel_part mainline
sunxi# run load_kernel
sunxi# run boot_kernel

After kernel boots, you can skip Debian configurations and drop the system to shell prompt.
